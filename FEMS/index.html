프레임 S-O-C-D-R — 판단을 구조화하는 언어

판단의 절차가 아니라, 내 판단에서 무엇이 빠졌는지를 확인하는 언어입니다.

SSituation왜 이 판단이 필요한가
OOptions어떤 대안을 검토하고 뭘 왜 버렸나
CCriteria무슨 기준으로 판단했나
DDecision무엇을 선택했고 재검토 조건은
RReflection결과가 나온 뒤 뭘 확인할 것인가

S-O-C-D-R × 데이터 분석 매핑

S-O-C-D-R마케팅 판단데이터 분석의 역할데이터가 보지 못하는 것
S 상황왜 이 판단이 필요한가기술통계·시각화로 현황 진단왜 ‘지금’ 문제인지의 맥락
O 선택지어떤 전략 대안이 있는가교차분석으로 기회/위험 발견조직 실행 가능성, 정치적 제약
C 기준무엇을 기준으로 판단하는가상관·회귀로 영향 변수 순위화정량으로 안 잡히는 가치 판단
D 결론어떤 전략을 선택하는가시계열·민감도로 시나리오 비교과거 ≠ 미래 보장 없음
실험맞는지 확인하려면A/B 테스트 설계실험 전까지는 가설
R 복기실행 후 돌아보기KPI 추적, 예측 vs 실적왜 그런 결과인지 해석

★ 네 번째 열(‘데이터가 보지 못하는 것’)이 오늘 하루 종일의 참조 카드입니다.

STEP 1 · 데이터 읽기 3문항

Q1. 이 데이터에서 “이건 그냥 넘기면 안 된다”고 느끼는 숫자나 패턴이 있습니까? (숫자+이유 1줄)
Q2. 데이터가 보여주는 것 중 예상과 가장 달랐던 것은 무엇입니까? (왜 달랐는지)
Q3. 이 패턴이 계속된다면 6개월 뒤 어떤 일이 벌어질까요? (긍정·부정 모두)
→ S 문장 완성: “______은 ______이다. 왜냐하면 __%이기 때문이다.”

STEP 3 · 레드팀 — 내 판단을 스스로 공격

공격 1 · 이 패턴이 이슈가 아닐 수 있다

계절 요인 아닐까? 샘플 수가 너무 적진 않나? 외부 충격(일회성) 아닐까?

공격 2 · 경영진의 반박

“그래서 뭘 어쩌라고?”에 데이터만으로 답할 수 없는 부분. 비용/실행주체/경쟁사 반응?

공격 3 · 틀리는 시나리오

“만약 ______라면, 내 판단은 틀린 것이다.” 6개월 뒤 틀렸다고 밝혀지는 상황.
공격이 잘 안 되면 → 판단이 탄탄. 공격이 너무 잘 되면 → S를 수정해야 합니다.

2막 6가지 눈 — S-O-C-D-R을 데이터로 채우기

각 눈이 S-O-C-D-R의 어느 칸을 채우는지 확인합니다.

1. 기술통계무슨 일이 일어났나? · R1 → S
2. 시각화패턴이 보이는가? · R1 → S
3. 교차분석누가, 어디서 다른가? · R2 → O
4. 상관·회귀얼마나 영향을 주는가? · R3 → C
5. 시계열·민감도앞으로 어떻게 될까? · R4 → D
6. 오류 검증AI가 틀리는 순간은? · 4막 → R

R3짧은 버전

마케팅 담당자가 바로 전략에 쓸 수 있는 언어로 설명해주세요.
지표해석 기준
R² (결정계수)0.7 이상이면 양호 · 독립변수가 종속변수 변동의 %를 설명
β (계수)부호와 크기 확인 · 1단위 변화 시 매출 변화량
p값 (유의확률)0.05 미만이면 통계적으로 유의
상관계수 r-1~+1 · |r|>0.7 강함 · |r|<0.3은 약한 상관
⚠️ r=0.3 미만은 ‘약한’ 상관입니다. AI가 ‘유의미’라 해도 반드시 크기를 확인하세요.

공통 해석 카드 — 매 라운드 결론 정리

총 4장(R1~R4) = 캡스톤의 재료. 네 번째 행(보이지 않는 것)이 핵심.

항목쓰는 것
데이터가 말해주는 것AI 결과를 한 문장으로
마케팅에서 의미하는 것내 사업/제품/고객 맥락에서의 해석
내가 할 수 있는 것구체적 액션 하나
이 분석이 보지 못하는 것한계 한 줄
S-O-C-D-R 중 어디에 쓰이나예: “C(기준)을 세우는 데 쓴다”

예시 (R1 기준)

데이터가 말해주는 것 — 월평균 매출 215,426천원, CV 33%로 변동이 크다. 12월이 최고(928,450), 2월이 최저(451,950).
마케팅에서 의미하는 것 — 매출이 계절에 크게 흔들린다. 비수기(1~2월) 매출 방어 전략이 필요.
내가 할 수 있는 것 — 1~2월 비수기 한정 프로모션 설계 / 비수기에 강한 제품군(젤리?) 밀어주기.
보지 못하는 것 — 왜 12월이 높은지(연말 선물? 성과급?) 원인 모름. 경쟁사도 같은 패턴인지 모름.
S-O-C-D-R — S(현황)를 데이터로 확인. “우리 매출은 계절 변동이 크고, 비수기 대응이 과제다.”

브릿지 A/B 테스트 — 상관에서 인과로

같은 조건에서 한 가지만 바꿔서 비교한다. A(기존) vs B(변경) → 차이가 우연인지 진짜인지 판단.

회귀분석: “관계가 있을 것 같다” → A/B 테스트: “진짜 그런지 확인했다”
S-O-C-D-R에서의 위치: D(결론)와 R(복기) 사이. “이 전략이 맞는지 확인하려면 A/B 테스트를 설계해야 한다.”

3막 AI가 그럴듯하게 틀리는 5가지 패턴

정교한 분석일수록 위험이 깊이 숨는다. 숫자만 보지 말고 해석과 결론도 보세요.

#유형AI의 실수 예시
#1해석 과장상관계수 -0.15(약한 상관)를 ‘유의미한 음의 상관’으로 서술
#2인과 비약프로모션 매출 -20%를 “저가 제품 때문”이라 단정, “중단하면 매출 상승” 비약
#3가짜 출처 (환각)「2024 과자류 시장 동향 보고서」— 실재하지 않는 자료를 자신있게 인용
#4서사 환각‘마트 쇼핑 선호’·‘자녀 간식 구매’ — 데이터에 없는 이유를 그럴듯하게 서술
#5보이지 않는 것 무시경쟁사·브랜드·마진 빠진 채 “추가 분석 없이 판단 가능” → 가장 위험
가장 위험한 오류는 숫자가 아니라 결론에 숨습니다. 오류를 찾는 것보다 중요한 건, 왜 못 찾았는가를 아는 것.

캡스톤 내 마케팅 이슈에 데이터를 입히기

오전에 쓴 이슈 + 2막 해석카드 4장 → 합쳐서 판단 메모 완성.

진행 (약 35분)

  1. 이슈 확정 + 분석 설계 · 5분
  2. AI 분석 + 결과 해석 · 10분
  3. S-O-C-D-R 판단 메모 작성 · 15분
  4. 발표 준비 · 5분

판단 메모 양식

S-O-C-D-R마케팅 판단데이터 분석 근거데이터가 보지 못하는 것
S 상황이슈 한 문장기술통계·시각화 현황왜 ‘지금’ 문제인지
O 선택지대안 + 배제 이유교차분석 기회/위험실행 가능성
C 기준판단 기준상관·회귀 변수 순위정량 밖 가치
D 결론전략 + 조건시나리오 비교과거≠미래
실험확인 방법A/B 설계 한 줄가설일 뿐
R 복기언제 뭘 확인KPI 추적해석은 사람
AI 협업뭘 시켰고, 뭘 반영/배제했는가
세 번째 열만 있으면 분석 보고서. 네 번째 열까지 있으면 판단 보고서.

발표 구조 (3분)

  • 30초 · S — 이슈 + 데이터 현황
  • 1분 · O · C — 기준으로 대안 비교
  • 1분 · D — 전략 결론 + 데이터 근거
  • 30초 · 보이지 않는 것 + 한계 (이 부분을 빼면 판단 발표가 아닙니다)
